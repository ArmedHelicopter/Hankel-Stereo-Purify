# 需求规格说明书 (PRD): Hankel-Stereo-Purify

## 1. 项目概述 (Project Overview)
本项目旨在开发一套基于多通道奇异谱分析 (MSSA) 的高保真音频降噪系统。针对 1950 年代早期模拟双声道录音（特定输入源：Jascha Heifetz, RCA "Living Stereo" FLAC 文件），系统需在滤除宽带高斯类底噪（Tape Hiss）的同时，严格保全原始音频的高频谐波结构与声道间绝对相位差。系统采用“前端参数探索 + 后端 CLI 批处理”的主从解耦架构，在保证工程验证效率的同时，严格控制物理计算与内存边界。

**实现说明（I/O 扩展）**：数据平面在 PRD 所述流式读取原则不变的前提下，允许通过 libsndfile 读写 **FLAC / WAV / AIFF / OGG** 等容器（见代码白名单）；验收与叙事仍以 **FLAC** 无损链路为主。

## 2. 数学模块与算法流水线 (Algorithm Pipeline)
本系统核心降噪逻辑由四个串行的数学模块构成。输入数据为离散时间序列 $X = (x_1, x_2, \dots, x_N)$，核心目标是通过空间投影实现非平稳主导信号与宽带高斯底噪的正交分离。

### 模块 A：Hankel 矩阵化模块 (Embedding & Hankelization)

* **功能定义：** 将一维音频时间序列映射为高维滞后相空间矩阵。
* **数学实现：** 设定窗口长度 $L$，列数 $K = N - L + 1$。构造矩阵 $H$，使得矩阵元素满足 $h_{i,j} = x_{i+j-1}$。该矩阵必须具有反斜对角线元素相等的 Hankel 结构特性。
* **输入/输出：**
    * 输入：一维序列 $X$ (长度 $N$)，标量 $L$。
    * 输出：二维矩阵 $H \in \mathbb{R}^{L \times K}$。

### 模块 B：MSSA 块矩阵组合模块 (Multichannel Block Construction)
* **功能定义：** 建立立体声（双声道）信号的空间相干性约束。
* **数学实现：** 分别对左声道序列 $X_{left}$ 和右声道序列 $X_{right}$ 执行模块 A，生成两个 Hankel 矩阵 $H_L$ 和 $H_R$。将其水平拼接，构造多通道块 Hankel 矩阵 $\mathbf{X}_{total}$：
    $$\mathbf{X}_{total} = [H_L, H_R] \in \mathbb{R}^{L \times 2K}$$

### 模块 C：SVD 正交分解与截断模块 (SVD & Subspace Truncation)

* **功能定义：** 求解块 Hankel 矩阵的主成分正交基，剥离噪声子空间。
* **数学实现：**
    1.  对 $\mathbf{X}_{total}$ 执行奇异值分解：$\mathbf{X}_{total} = U \Sigma V^T$。
    2.  提取对角矩阵 $\Sigma$ 中的奇异值 $\sigma_i$，计算各分量的累计能量贡献率。
    3.  设定截断秩 $k$。保留前 $k$ 个对应于确定性主导信号（如乐器基频与谐波）的奇异值，将剩余对应于随机底噪的奇异值置零，生成截断奇异值矩阵 $\Sigma_k$。
    4.  计算降噪后的低秩近似矩阵：$\mathbf{\hat{X}}_{total} = U \Sigma_k V^T$。
* **输入/输出：**
    * 输入：二维块矩阵 $\mathbf{X}_{total}$，标量 $k$。
    * 输出：降维滤波后的低秩二维块矩阵 $\mathbf{\hat{X}}_{total}$。

### 模块 D：对角平均化重构模块 (Diagonal Averaging)
* **功能定义：** 将降噪后的低秩近似块矩阵逆向映射回一维时间序列。
* **数学实现：** 由于截断重构后的矩阵 $\mathbf{\hat{X}}_{total}$ 破坏了严格的 Hankel 结构，必须沿其反斜对角线对元素求平均值。对于子矩阵中的每一项，通过计算 $\hat{x}_n = \frac{1}{|S_n|} \sum_{(i,j) \in S_n} \hat{h}_{i,j}$ （其中 $S_n$ 为第 $n$ 条反角线上的元素坐标集合），重构出滤波后的离散序列。
* **输入/输出：**
    * 输入：低秩二维块矩阵 $\mathbf{\hat{X}}_{total}$。
    * 输出：降噪后的左声道一维序列 $\hat{X}_{left}$ 与右声道一维序列 $\hat{X}_{right}$。

## 3. 功能需求 (Functional Requirements)
* **F-01 [I/O 管线与读取机制]:** 原生支持超大体积（≥ 300MB）FLAC 无损格式的读取。严禁在物理磁盘层面切割原文件，必须采用软件级动态文件指针偏移机制（如 `soundfile.blocks`），执行原地顺序读取，并将 PCM 数据覆盖写入定长内存缓冲区。
* **F-02 [短时平稳分帧]:** 实现基于重叠相加法（Overlap-Add）的流式分帧架构。支持自定义帧长（Frame Size）与跳步（Hop Size）。重构阶段必须通过加窗机制（如 Hanning 窗）执行交叉平滑，消除帧间截断误差。

* **F-03 [相位锁定降噪]:** 强制通过模块 B 构建联合矩阵分解，避免左右声道独立执行 SVD 引发的特征空间错位与声像漂移。
* **F-04 [动态子空间界定]:** 提供能量贡献率接口，支持硬截断秩 $k$ 的动态输入以切分信号与噪声。
* **F-05 [可选/二期规划 - 交互式前端控制台与任务下发]:** 构建基于本地轻量级框架（如 Streamlit）的控制平面。该需求非 MVP（最小可行性产品）核心环节。
    1. **参数寻优 (EDA)：** 支持加载微型样本切片。为控制开发成本，系统**不包含**自动噪声寻优算法，仅通过在底层指定固定的读取偏移量（如强制读取文件前 5 秒）来捕获预备期的背景底噪，并实时渲染降噪前后的对比频谱。
    2. **全量任务指定与派发：** 提供本地文件路径指定组件（文本输入框或调用操作系统的本地文件选择器），允许用户直接指定待处理超大原文件的绝对路径，并提供一键执行按钮以触发后台全量批处理。

* **实现备注（当前仓库）：** 原需求为可选控制平面；现已在仓库根目录提供 **Streamlit** 实现 [`frontend/app.py`](../frontend/app.py)（安装见 `requirements-frontend.txt`）。**MVP 主路径仍为 CLI**；EDA 与全量任务派发行为以 README / 前端页内说明为准。

## 4. 非功能性需求与系统边界 (Non-Functional Requirements)
* **NF-01 [内存隔离红线]:** 全量数据流式处理期间，系统常驻内存峰值 (Peak RSS) 必须严格控制在 2GB 以内，彻底杜绝 OOM 崩溃异常。
* **NF-02 [零拷贝算子与基准对齐]:** 严禁使用高层显式循环构建轨迹矩阵。必须调用 `numpy.lib.stride_tricks.as_strided` 实现底层步长的零拷贝修改。该模块由 CPU 串行执行，作为最高精度的黄金基准模型（Golden Model），为底层数字集成电路阵列提供测试向量验证标准。
* **NF-03 [物理声学极限]:** 算法容差要求：重构后音频的声道绝对相位响应差趋近于零；10kHz 以上的高次谐波能量在频域检验中无显著数学衰减。
* **NF-04 [异步流水线架构]:** 单帧微观运算（A 至 D）保持同步阻塞，遵循 RAW 读写依赖；宏观系统预留生产者-消费者（Producer-Consumer）架构接口，以便后续解耦磁盘 I/O 预取与 CPU 高密度矩阵分解。
* **NF-05 [可选/二期规划 - 主从进程解耦与内存物理隔离]:** 若实现 F-05 的前端界面，严禁前端框架直接读取、加载或网络传输全量大文件。前端必须通过操作系统子进程调用（如 `subprocess.Popen`），将前端获取的文件绝对路径及参数（$L, k$）传递给底层的 CLI 数据平面。通过分离模式（Detached Mode）唤醒后端流水线独立执行，确保图形界面开销与核心矩阵运算在系统内存空间上实现绝对的物理隔离。

* **实现备注（当前仓库）：** 现有 [`frontend/app.py`](../frontend/app.py) 全量批处理使用 `subprocess.run` 调用 `python -m src.cli`（参数列表，无 `shell=True`），与上述隔离要求一致。

## 5. 验收标准 (Acceptance Criteria)
1. 成功流式读取并静默重构测试大文件（如 1.6GB），无需依赖任何图形界面。
2. 若启用了二期前端，EDA 看板能在一键调整参数后，5 秒内完成频谱重渲染，并成功唤醒后台进程处理大文件。
3. 挂载性能探针证实核心 CLI 处理全周期的物理内存占用始终被压制在 2GB 水位线以下。
4. SVD 截断误差率与模块 D 重构残差收敛至预期容差域内。
