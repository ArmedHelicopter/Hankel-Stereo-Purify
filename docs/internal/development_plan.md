# 开发计划与协作风险管理

> **适用性**：本文为**历史协作与里程碑规划**，非 Phase0 工程基线必读。**当前代码组织与接口以 `tutorial` 分支及 [`software_design.md`](../software_design.md) 为准**（单帧 `process_frame` 链、无 `MSSAStage`/Pipeline 调度器）。当前 **`main`** 可能仅为骨架。环境与分支见 [`SETUP_AND_BUILD.md`](../SETUP_AND_BUILD.md)、[`BRANCHES.md`](../BRANCHES.md)。

## 目标

为 `Hankel-Stereo-Purify` 设定一个清晰的一月交付计划，包含三人协作分工、每周里程碑，以及冲突风险与缓解方案。

---

## 1. 项目总体拆分

### 1.1 核心模块

- `src/io`：音频 I/O 与流式预处理
- `src/core`：MSSA 算法流水线与数学模块
- `src/facade`：顶层接口与 CLI 集成
- 旧顶层路径（如 `src/audio_utils.py`、`src/hankel_matrix.py`、`src/mssa_core.py`）已不再采用，项目已统一新 `src/` 结构路线

### 1.2 关键设计原则

- 先“做可运行 MVP”再“做抽象优化”
- 保持高内聚、低耦合，接口契约优先
- 以 **单帧 `process_frame` 链**、**截断配置类型**与 **`AudioPurifier` 门面**为主线，避免过度设计（详见 [`software_design.md`](../software_design.md)）

验收：

- 开始输出至少对齐的中间数据格式

### 第 2 周：核心算法实现

目标：完成 A/B/C/D 算法模块与基本策略实现

- 实现 `d_diagonal.py` 的对角重构逻辑
- 完成 `FixedRankStrategy`
- 完成 `HanningWindowStrategy`

验收：

- 重构输出长度与期望一致

### 第 3 周：性能优化与大文件验证

目标：完成大文件流式处理验证与内存控制

验收：
- 目标文件可以运行
- 内存峰值控制在 2GB 左右
- 不出现重大维度/IO 错误

### 第 4 周：稳定发布与交付

目标：完成 CLI、测试、文档和最终验收

## 4. 冲突风险与缓解方案

### 风险 1：目录结构与实现方式不一致

- 原因：`src/` 同时存在旧结构与新 `src/core/...` 结构
- 影响：团队成员可能在不同路径上重复实现，造成冲突和浪费

缓解：
- 立刻统一主导目录结构，明确“主开发路径”
- 将旧顶层空模块设为过渡文件或删除
- 在每日立会上确认各人当前工作文件列表

### 风险 2：接口契约不明确导致联调失败

- 原因：A/B/C 各模块未提前定义好输入/输出格式
- 影响：集成时花费大量时间调整数据格式

缓解：
- 在第 1 周完成接口文档，明确 `process_frame` 单帧张量契约与 `read_blocks()` 数据结构
- 成员间用“假数据快速契约测试”验证格式
- 采用简单的 JSON / 文本说明接口约定

### 风险 3：过度抽象影响交付速度

- 原因：投入太多精力先做泛型调度层或复杂建造者，而非先做可运行单帧链
- 影响：一月内可能无法完成核心功能

缓解：
- 采用“先可运行，再抽象”的开发策略
- 将设计模式当成后期重构目标，而不是先行目标
- 仅实现最小可行的接口，后续再补策略扩展

### 风险 4：大文件性能瓶颈变成交付阻碍

- 原因：SVD 与大矩阵处理在 300MB+ 情况下可能耗时、内存暴涨
- 影响：交付延迟、无法满足非功能要求

缓解：
- 第 2 周结束前完成小文件可运行后，立即开始大文件测试
- 采用分阶段验证：先验证逻辑正确，再验证性能约束
- 预留时间在第 3 周做内存/性能调优

### 风险 5：团队同步不足导致重复工作

- 原因：三人同时开发不同模块但未及时共享进展
- 影响：接口重复实现、编码方向不一致

缓解：
- 每天 10-15 分钟同步进展
- 每周一次联调检查
- 重要接口与问题写入一个共享文档或议题列表

---

## 5. 交付建议

- 先以 `src/facade/AudioPurifier` + `src/cli.py` 作为最终可交付产物
- 扩展能力（非本月核心）在交付后再单独排期
- 在交付时保证“一个可运行示例 + 一份性能验证结果”

---

> 备注：该计划适用于三人团队、一个月交付周期，优先保证“核心算法可运行、接口明确、性能可控”。
